JavaScript设计模式-建造者模式

发表于 2018-07-03 | 更新于 2018-07-04 | 分类于 JavaScript设计模式 | 评论数：

建造者模式(Builder)：将一个复杂对象的构建层与其表示层相互分离，同样的构建过程可采用不同的表示。

特点

  • 注重对象创建过程。（通常将创建对象的类模块化，使得被创建的类的每一个模块都可以得到灵活与高质量的复用）

使用场景

  • 需要生成的对象具有复杂的内部结构
  • 需要生成的对象内部属性本身相互依赖。

优点

  • 封装性好。使用建造者模式可以有效的封装变化，在使用建造者模式的场景中，一般产品类和建造者类是比较稳定的，因此，将主要的业务逻辑封装在指导者类中对整体而言可以取得比较好的稳定性。
  • 易扩展。新的需求，可以通过实现一个新的建造者类即可

缺点

  • 增加了结构的复杂性。
