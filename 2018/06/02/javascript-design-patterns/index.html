学习Javascript设计模式

发表于 2018-06-02 | 更新于 2018-07-07 | 分类于 JavaScript设计模式 | 评论数：

在日常的学习生活中，我越来越觉得有必要好好学习设计模式了。

  • 想深入了解面向对象编程思想
  • 自己的模块化开发能力有待提高
  • 希望自己能写出高效率、可拓展、可维护、一目了然、结构清晰的代码

所以我决定花一段时间，通过《JavaScript设计模式》、《JavaScript设计模式与开发实践》这两本书以及网上的一些资料，来达成以上目标

何为设计模式？

设计模式是在面向对象软件设计设计过程中针对特定问题的简洁而优雅的解决方案。通俗一点说，设计模式是在某种场合下对某个问题的一种解决方案。

学习设计模式的作用

在软件设计中，模式是一些经过了大量实际项目验证的优秀解决方案。在你熟悉这些设计模式后，当合适的场景出现时，就可以很快地找到某种模式作为解决方案。

设计模式的适用性

从某些角度来看，设计模式确实有可能带来代码量的增加，或许也会把系统的逻辑搞得更复杂。但是软件开发的成本并非全部在开发阶段，设计模式的作用是让人们写出可复用和可维护性高的程序。

所有设计模式的实现都遵循一条原则，即“找出程序中变化的地方，并将变化封装起来”。

一个程序的设计总是可以分为可变的部分和不变的部分。当我们找出可变的部分，并把这些部分封装起来，那么剩下的就是不变和稳定的部分。这些不变和稳定的事非常容易复用的。这也是设计模式为什么描写的事可复用面向对象软件基础的原因。

为了避免设计模式被误用和滥用，我们应该在深刻理解模式的意图之后，再结合项目的实际场景，将其用在正确的地方。

学会分辨模式

分辨模式的关键是意图而不是结构。

许多模式的类图看起来都差不多(如代理模式和装饰者模式、策略模式和状态模式、策略模式和智能命令模式等)，模式只有放在具体的环境下才有意义（如手机，把它当电话的时候，它就是电话；把它当闹钟的时候，它就是闹钟；用它玩游戏的时候，它就是游戏机）。

辨别模式的关键是这个模式出现的场景，以及为我们解决了什么问题。

设计模式类型

创建型

创建型设计模式：是一类处理对象创建的设计模式，通过某种方式控制对象的创建来避免基本对象创建时可能导致的设计上的问题或增加设计上的复杂度。

  • 简单工厂模式
  • 工厂方法模式
  • 抽象工厂模式
  • 建造者模式
  • 原型模式
  • 单例模式

结构型

结构型设计模式：关注于如何将类或对象组合成更大、更复杂的结构，以简化设计。

  • 外观模式
  • 适配器模式
  • 代理模式
  • 装饰者模式
  • 桥接模式
  • 组合模式
  • 享元模式

行为型

行为型设计模式：用于不同对象之间职责划分或算法抽象。该设计模式不仅涉及类和对象，还涉及类或对象之间的交流模式并加以实现。

  • 模板方法模式
  • 观察者模式
  • 状态模式
  • 策略模式
  • 职责链模式
  • 命令模式
  • 访问者模式
  • 中介者模式
  • 备忘录模式
  • 迭代器模式
  • 解释器模式

技巧型

技巧型设计模式：是通过一些特定技巧来解决组件的某些方面的问题，这些技巧一般通过实践经验总结得到。

  • 链模式
  • 委托模式
  • 数据访问对象模式
  • 节流模式
  • 简单模板模式
  • 参与者模式
  • 等待者模式

架构型

架构型设计模式：是一类框架结构，通过提供一些子系统，指定它们的职责，并将它们的条理清晰地组织在一起。

  • 同步模块模式
  • 异步模块模式
  • Widget 模式
  • MVC 模式
  • MVP 模式
  • MVVM 模式

参考

  • 《JavaScript设计模式与开发实践》 – 曾探
